## 四、当前推荐使用方式

当前建议按下面顺序工作：

1. 在本地修改代码
2. 同步到挂载后的远端仓库
3. 用 DLP 提交远端训练
4. 在远端工作目录确认 best checkpoint
5. 将 best 权重拉回本地 `checkpoints/`
6. 使用本地可视化脚本做连续样本和离散样本复查

## 五、当前主要短板

从现阶段结果看，主要问题不在“链路跑不通”，而在“模型还不够稳”。

比较明确的短板有：

- 连续样本指标看起来可以，但离散样本误差明显飙升
- 当前 best 只是在已有训练设置下的阶段性 best，不代表全局最优
- 训练后期出现性能回退，说明现有训练策略还不够稳
- 本地目前主要验证了 CPU 推理，尚未把 MPS 当成正式支持路径
