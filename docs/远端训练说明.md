# 远端训练说明

## 文档目的

本文档说明如何在 macOS 本地通过挂载 NFS 与 DLP 平台，启动
`Person-in-WiFi-3D-repo` 的远端训练。

本流程已经基于当前项目结构和本次排障结果整理完毕，可直接复用。

当前文档以“已经打通并验证过的链路”为准，不再保留已经废弃的尝试路径。

## 一、整体思路

本地机器只负责：

- 挂载远端 NFS
- 准备远端可见的仓库与数据
- 提交 DLP 任务
- 查看日志与同步结果

真正训练发生在远端 Linux 容器中。

## 二、目录约定

推荐使用以下路径：

- 远端 NFS 根目录：`/cloud/tmp/yc`
- 远端仓库目录：`/cloud/tmp/yc/Person-in-WiFi-3D-repo`
- 远端数据目录：`/cloud/tmp/yc/Person-in-WiFi-3D-repo/data/wifipose`

本地挂载后对应：

- 本地挂载点：`/Users/yc1/Documents/yc`
- 本地仓库映射目录：`/Users/yc1/Documents/yc/Person-in-WiFi-3D-repo`

## 三、在 macOS 上挂载远端 NFS

示例命令：

```bash
sudo mount -t nfs -o nolock,resvport,rw 172.30.2.93:/cloud/tmp/yc /Users/yc1/Documents/yc
```

挂载成功后：

- 本地对 `/Users/yc1/Documents/yc/...` 的读写
- 等价于远端对 `/cloud/tmp/yc/...` 的读写

## 四、准备远端仓库

需要确保远端能直接看到仓库代码：

- 本地实际开发仓库：`/Volumes/Work/Person-in-WiFi-3D-repo`
- 远端训练使用仓库：`/Users/yc1/Documents/yc/Person-in-WiFi-3D-repo`

建议做法：

1. 本地修改 `/Volumes/Work/Person-in-WiFi-3D-repo`
2. 将改动后的关键文件同步到挂载目录中的同路径仓库
3. DLP 任务始终使用挂载目录对应的远端仓库副本

## 五、准备训练数据

远端训练脚本默认要求数据位于：

- `/cloud/tmp/yc/Person-in-WiFi-3D-repo/data/wifipose`

数据目录结构必须满足：

- `train_data/csi/*.mat`
- `train_data/keypoint/*.npy`
- `train_data/train_data_list.txt`
- `test_data/csi/*.mat`
- `test_data/keypoint/*.npy`
- `test_data/test_data_list.txt`

训练前建议：

- 清理 macOS 产生的 `._*` 文件
- 确保 list 文件与样本数量匹配

当前远端训练脚本会自动删除数据目录中的 `._*` 文件。

## 六、远端训练入口

提交训练任务使用：

- `remote_platform/dlp/dlpflow_train.py`

该脚本会生成 DLP 任务，并在远端执行：

- `remote_platform/dlp/run_remote_train.sh`

## 七、远端训练脚本实际做的事

`run_remote_train.sh` 的执行顺序如下：

1. 设置代理环境变量
2. 进入远端仓库目录
3. 设置 `PYTHONPATH`
4. 设置 `TORCH_HOME` 与 `TORCH_EXTENSIONS_DIR`
5. 删除数据目录中的 `._*`
6. 按需安装依赖：
   - `remote_platform/dlp/requirements-remote-base.txt`
   - `third_party/mmcv`
   - `third_party/mmdet`
   - 当前仓库本身
7. 运行数据校验脚本
8. 用 `tools/train.py` 启动训练

## 八、数据校验逻辑

训练前会调用：

- `remote_platform/dlp/validate_remote_dataset.py`

它会检查：

- 训练集和测试集目录是否存在
- `train_data_list.txt` 和 `test_data_list.txt` 是否存在
- `.mat` 与 `.npy` 是否非空

如果校验失败，训练不会启动。

## 九、关键环境变量

常用变量如下：

- `DLP_REMOTE_NFS_PATH`
- `DLP_REMOTE_REPO_DIR`
- `PIWIFI_DATASET_ROOT`
- `PIWIFI_WORK_DIR`
- `PIWIFI_REMOTE_PYTHON`
- `PIWIFI_REMOTE_INSTALL_DEPS`
- `PIWIFI_SAMPLES_PER_GPU`
- `PIWIFI_WORKERS_PER_GPU`
- `PIWIFI_CFG_OPTIONS`


## 十、全量训练示例

下面是“全数据训练 20 轮，并按 `mpjpe` 保存最佳权重”的提交示例：

```bash
DLP_REMOTE_NFS_PATH=/cloud/tmp/yc \
DLP_REMOTE_REPO_DIR=/cloud/tmp/yc/Person-in-WiFi-3D-repo \
PIWIFI_DATASET_ROOT=/cloud/tmp/yc/Person-in-WiFi-3D-repo/data/wifipose \
PIWIFI_REMOTE_PYTHON=python3 \
PIWIFI_REMOTE_INSTALL_DEPS=1 \
PIWIFI_WORK_DIR=/cloud/tmp/yc/Person-in-WiFi-3D-repo/work_dirs/remote_train_full_resume20epochs_20260406 \
PIWIFI_SAMPLES_PER_GPU=2 \
PIWIFI_WORKERS_PER_GPU=0 \
PIWIFI_SEED=3407 \
PIWIFI_DETERMINISTIC=1 \
PIWIFI_VAL_MAX_SAMPLES=512 \
PIWIFI_CFG_OPTIONS="runner.max_epochs=20 evaluation.interval=5 evaluation.save_best=mpjpe evaluation.rule=less checkpoint_config.interval=5 checkpoint_config.max_keep_ckpts=10 log_config.interval=20" \
conda run -n sam-dlp python /Volumes/Work/Person-in-WiFi-3D-repo/remote_platform/dlp/dlpflow_train.py
```

说明：

- `runner.max_epochs=20` 表示训练 20 轮
- `evaluation.interval=5` 表示每 5 轮验证一次
- `evaluation.save_best=mpjpe` 表示按 `mpjpe` 记录最佳模型
- `evaluation.rule=less` 表示 `mpjpe` 越小越好
- `PIWIFI_VAL_MAX_SAMPLES=512` 表示验证阶段只抽取前 512 个测试样本

当前这一路径已经实际跑完，最佳权重出现在第 `10` 轮：

- `best_mpjpe_epoch_10.pth`
- `mpjpe=388.7639`

## 十一、5 样本训练测试示例

如果要做 5 样本冒烟测试，可以保留真实数据目录不变，只覆盖训练清单：

```bash
DLP_REMOTE_NFS_PATH=/cloud/tmp/yc \
DLP_REMOTE_REPO_DIR=/cloud/tmp/yc/Person-in-WiFi-3D-repo \
PIWIFI_DATASET_ROOT=/cloud/tmp/yc/Person-in-WiFi-3D-repo/data/wifipose \
PIWIFI_REMOTE_PYTHON=python3 \
PIWIFI_REMOTE_INSTALL_DEPS=1 \
PIWIFI_WORK_DIR=/cloud/tmp/yc/Person-in-WiFi-3D-repo/work_dirs/remote_train_5samples_manifest_fix8_scatter_20260406 \
PIWIFI_SAMPLES_PER_GPU=2 \
PIWIFI_WORKERS_PER_GPU=0 \
PIWIFI_CFG_OPTIONS="runner.max_epochs=8 checkpoint_config.interval=1000 evaluation.interval=1000 log_config.interval=1 lr_config.step=[]" \
PIWIFI_TRAIN_LIST_FILE=/cloud/tmp/yc/Person-in-WiFi-3D-repo/remote_platform/dlp/manifests/train_5samples.txt \
conda run -n sam-dlp python /Volumes/Work/Person-in-WiFi-3D-repo/remote_platform/dlp/dlpflow_train.py
```

这个模式适用于：

- 检查训练链路是否能真正跑起来
- 验证 loss 是否下降
- 在不跑全量数据的情况下快速定位错误

## 十二、验证与最佳权重策略

项目基础配置位于：

- `configs/wifi/petr_wifi.py`

远端训练当前实际使用的是：

- `configs/wifi/petr_wifi_remote.py`

当前建议的正式训练策略是：

- `runner.max_epochs=20`
- `evaluation.interval=5`
- `evaluation.save_best='mpjpe'`
- `evaluation.rule='less'`
- `checkpoint_config.interval=5`
- `checkpoint_config.max_keep_ckpts=10`
- `PIWIFI_VAL_MAX_SAMPLES=512`

这表示：

- 每 5 个 epoch 做一次验证
- 验证集默认不跑全量，而是只跑前 `512` 个样本
- 每 5 个 epoch 存一次权重
- 额外保留当前最佳 `mpjpe` 权重
- `mpjpe` 越小越好

## 十三、如何看日志

查看任务列表：

```bash
conda run -n sam-dlp dlpctl get jobs -o table
```

查看标准输出：

```bash
conda run -n sam-dlp dlpctl logs task job-<id>-worker-0 -s stdout
```

查看标准错误：

```bash
conda run -n sam-dlp dlpctl logs task job-<id>-worker-0 -s stderr
```

通常可以这样判断进度：

- 出现 `remote_dataset_ok`：数据校验通过
- 出现 `Start running`：runner 已经正式启动
- 出现 `Epoch [x][y/z]`：训练迭代已开始
- 出现 `Saving checkpoint`：checkpoint 已经落盘

如果本地 `dlpctl` 访问需要代理，可以先导出：

```bash
export http_proxy="http://p.truesightai.com:1087"
export https_proxy="http://p.truesightai.com:1087"
```

## 十六、推荐使用流程

建议按以下顺序操作：

1. 在本地修改代码
2. 将关键改动同步到挂载目录中的远端仓库副本
3. 确认数据目录已就位
4. 先跑 5 样本冒烟测试
5. 确认训练 loss 能下降
6. 再跑带验证的全量短任务
7. 确认 `mpjpe` 与最佳模型保存策略正常后，再启动正式全量训练

## 十六、训练完成后应查看哪些结果

训练完成后重点查看 `work_dir` 下的内容：

- 日志文件
- `epoch_*.pth`
- `latest.pth`
- 若开启 `save_best`，则查看最佳模型相关文件或元信息

这些结果可以再通过挂载目录同步回本地，用于：

- 本地可视化
- 本地评估
- 二次分析
